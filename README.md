# VERBTW FILMS

Личный киноархив VERBTW с просмотренными фильмами и сериалами. В каталоге можно найти нужную картину, открыть отдельную страницу, посмотреть трейлер прямо на сайте и поставить собственную оценку.

**Рабочая версия:** [verbtw-films.vercel.app](https://verbtw-films.vercel.app)

## Что умеет сайт

- каталог из 312 просмотренных фильмов и сериалов;
- поиск по названию;
- фильтры по типу контента, жанру и режиссёру;
- сортировка по оценке IMDb по возрастанию и убыванию;
- обложки, описания, режиссёры и IMDb-оценки из открытых источников;
- отдельная страница для каждого фильма или сериала;
- предпросмотр трейлера с YouTube прямо на сайте при наведении;
- регистрация и вход по электронной почте;
- личные оценки от 1 до 10, синхронизируемые между устройствами;
- общий рейтинг VERBTW на основе оценок зарегистрированных пользователей;
- возврат к прежней карточке, позиции прокрутки, поиску и фильтрам после оценки;
- адаптивная версия для компьютеров и телефонов;
- переключение русского и английского языков с сохранением выбора.

## Технологии

- HTML, CSS и JavaScript без фронтенд-фреймворков;
- Python для локального сервера и API-функций;
- Supabase Auth и PostgreSQL для аккаунтов и оценок;
- Vercel Functions для API;
- Vercel для хостинга и production-деплоев.

## Структура проекта

| Путь | Назначение |
| --- | --- |
| `index.html`, `app.js`, `styles.css` | главная страница, каталог, поиск, фильтры и сортировка |
| `film.html`, `film.js`, `film.css` | страница отдельного фильма, трейлер и выставление оценки |
| `data.js` | список просмотренных фильмов и сериалов |
| `community.js`, `community.css` | аккаунты, личные оценки и рейтинг сообщества |
| `api/config.py` | публичная конфигурация подключения к Supabase |
| `api/imdb.py` | поиск IMDb ID и получение оценки |
| `api/trailer.py` | поиск трейлера на YouTube без загрузки видео на сервер |
| `supabase/schema.sql` | таблицы, политики безопасности, триггер профиля и функция общего рейтинга |
| `server.py` | локальный HTTP-сервер |
| `vercel.json` | настройки функций Vercel |

## Локальный запуск

Требуется Python 3. Запусти из корня проекта:

```bash
python3 server.py
```

После запуска открой [http://localhost:8000](http://localhost:8000).

Каталог, страницы фильмов и поиск трейлеров работают через локальный сервер. Аккаунты и облачные оценки требуют подключения Supabase.

## Настройка Supabase

1. Создай проект в Supabase.
2. Открой SQL Editor и выполни содержимое `supabase/schema.sql`.
3. Добавь в локальное окружение или настройки Vercel публичные переменные:

```env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_PUBLISHABLE_KEY=your_publishable_key
```

Вместо `SUPABASE_PUBLISHABLE_KEY` также поддерживается `SUPABASE_ANON_KEY`. Варианты с префиксом `NEXT_PUBLIC_` тоже распознаются.

Приватные ключи `service_role`, JWT secret и другие административные данные нельзя добавлять в репозиторий или клиентский код.

## Как устроены оценки

Пользовательские оценки хранятся в таблице `ratings` и привязаны к аккаунту. Политики Row Level Security разрешают пользователю изменять только собственные записи. Публичная функция `get_rating_summaries()` возвращает среднюю оценку и количество голосов без раскрытия данных пользователей.

Если человек ещё не вошёл в аккаунт, выбранная оценка временно сохраняется в браузере и записывается в базу после авторизации.

## Добавление нового фильма

1. Добавь русское название в конец массива `movies` в `data.js`.
2. Если название неоднозначное, добавь точную страницу Википедии в `wikiTitleOverrides` внутри `app.js` и `film.js`.
3. Если IMDb определяет другую картину, добавь правильный IMDb ID в `IMDB_ID_OVERRIDES` внутри `api/imdb.py`.
4. При необходимости уточни поисковое название трейлера в `trailerTitleOverrides` внутри `film.js`.
5. Проверь синтаксис:

```bash
node --check data.js
node --check app.js
node --check film.js
python3 -m py_compile api/imdb.py api/trailer.py api/config.py
```

## Источники данных

- Википедия и Wikidata — обложки, описания и режиссёры;
- IMDb/Cinemeta — IMDb ID, год и рейтинг;
- YouTube — встраиваемые трейлеры.

Видео не копируются и не хранятся в проекте: сайт встраивает доступный ролик с YouTube.
